JavaScript Modules

1. What are Modules?

  • Modules allow breaking code into separate files for better organization.
  • Each module can export functionality to be used in other files.
Example Preview:
Modules help keep JavaScript code modular and maintainable.

2. Exporting

  • Use export keyword to export variables, functions, or classes.
Example:

export const name = "Alice";

export function greet() {
  console.log("Hello, " + name);
}
Output Preview:
Exported variables or functions can be imported in another file.

3. Importing

  • Use import keyword to access exported items from other modules.
Example:

import { name, greet } from './module.js';

console.log(name); // Alice
greet();         // Hello, Alice
Output Preview:
Alice
Hello, Alice

4. Default Export

  • Each module can have one default export.
Example:

export default function greet() {
  console.log("Hello from default export");
}

import greetFunc from './module.js';
greetFunc(); // Hello from default export
Output Preview:
Hello from default export

5. Named Exports

  • Export multiple items from a module using named exports.
Example:

export const firstName = "Alice";
export const lastName = "Smith";

import { firstName, lastName } from './module.js';
console.log(firstName + " " + lastName); // Alice Smith
Output Preview:
Alice Smith
