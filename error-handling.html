JavaScript Error Handling

1. What is Error Handling?

  • Error handling allows you to manage runtime errors gracefully.
  • Prevents your program from crashing unexpectedly.
Example Preview:
Try/catch helps handle exceptions and show user-friendly messages.

2. try / catch

  • Wrap code in try to detect errors.
  • Handle errors in catch.
Example:

try {
  let result = nonExistentFunction();
} catch (error) {
  console.log("Error caught:", error.message);
}
Output Preview:
Error caught: nonExistentFunction is not defined

3. finally

  • finally block executes regardless of error.
Example:

try {
  console.log("Inside try");
} catch (err) {
  console.log("Inside catch");
} finally {
  console.log("Finally always runs");
}
Output Preview:
Inside try
Finally always runs

4. Throwing Custom Errors

  • Use throw to create custom error messages.
Example:

function checkAge(age) {
  if(age < 18) {
    throw new Error("Age must be 18 or older");
  }
  return "Access granted";
}

try {
  console.log(checkAge(15));
} catch (err) {
  console.log("Error:", err.message);
}
Output Preview:
Error: Age must be 18 or older

5. Handling Async Errors

  • Use try/catch with async/await to handle errors in asynchronous code.
Example:

async fetchData() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/invalid");
    const data = await response.json();
    console.log(data);
  } catch (err) {
    console.log("Async Error:", err);
  }
}

fetchData();
