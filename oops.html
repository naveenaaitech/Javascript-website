JavaScript OOP

1. Classes & Objects

  • Class: Blueprint for objects.
  • Object: Instance of class with data and behavior.
Example:

class Person {
  constructor(name, age) { this.name = name; this.age = age; }
  greet() { console.log("Hello, " + this.name); }
}
const p1 = new Person("Alice", 25);
p1.greet();
Output Preview:
Hello, Alice

2. Inheritance

  • Inheritance: Child class can reuse parent class properties & methods.
  • Single inheritance: supported using extends.
  • Multi-level inheritance: class can inherit from a class which inherits from another.
  • Hierarchical inheritance: multiple subclasses can inherit from the same parent.
  • Multiple inheritance: not supported natively. Use mixins to simulate.
  • Hybrid inheritance: combination of multi-level + multiple inheritance using classes and mixins.
Example – Single Inheritance:

class Person {
  constructor(name, age) { this.name = name; this.age = age; }
  greet() { console.log("Hello, " + this.name); }
}

class Student extends Person {
  constructor(name, age, grade) {
    super(name, age);
    this.grade = grade;
  }
}
const s1 = new Student("Bob", 20, "A");
console.log(s1.name, s1.grade);
Output Preview:
Bob A
Example – Multi-level Inheritance:

class Graduate extends Student {
  constructor(name, age, grade, degree) {
    super(name, age, grade);
    this.degree = degree;
  }
}
const g1 = new Graduate("Alice", 23, "A", "MSc");
console.log(g1.name, g1.grade, g1.degree);
Output Preview:
Alice A MSc
Example – Hierarchical Inheritance:

class Teacher extends Person {
  constructor(name, age, subject) {
    super(name, age);
    this.subject = subject;
  }
}
const t1 = new Teacher("Mr. Smith", 40, "Math");
console.log(t1.name, t1.subject);
Output Preview:
Mr. Smith Math
Example – Multiple Inheritance (Mixin):

const canEat = { eat() { console.log(this.name + " is eating"); } };
const canStudy = { study() { console.log(this.name + " is studying"); } };

class Learner { constructor(name) { this.name = name; } }
Object.assign(Learner.prototype, canEat, canStudy);

const l1 = new Learner("Charlie");
l1.eat();
l1.study();
Output Preview:
Charlie is eating
Charlie is studying
Example – Hybrid Inheritance (Multi-level + Mixins):

const canWalk = { walk() { console.log(this.name + " is walking"); } };

class PostGrad extends Graduate {}
Object.assign(PostGrad.prototype, canWalk);

const pg1 = new PostGrad("David", 25, "A+", "PhD");
pg1.greet();  // from Person
console.log(pg1.name, pg1.grade, pg1.degree);
pg1.walk();   // from Mixin
Output Preview:
Hello, David
David A+ PhD
David is walking

3. Polymorphism

  • Polymorphism = ability to take many forms.
  • Method Overriding: supported (child replaces parent method).
  • Method Overloading: not supported natively. Can simulate using default params or arguments.
Example – Overriding:

class Person { greet() { console.log("Hello from Person"); } }
class Student extends Person { greet() { console.log("Hello from Student"); } }
const s = new Student();
s.greet();
Output Preview:
Hello from Student
Example – Simulating Overloading:

class Calculator {
  add(a, b, c) {
    if(a && b && c) return a + b + c;
    else if(a && b) return a + b;
    else return a;
  }
}
const calc = new Calculator();
console.log(calc.add(2));     
console.log(calc.add(2,3));   
console.log(calc.add(2,3,4));
Output Preview:
2
5
9

4. Encapsulation

  • Encapsulation: restrict direct access to object properties.
  • Private fields: # supported (ES2022+).
Example:

class Person { #ssn;
  constructor(name, ssn) { this.name = name; this.#ssn = ssn; }
  getSSN() { return this.#ssn; }
}
const p1 = new Person("Alice", "123-45-6789");
console.log(p1.name);    
console.log(p1.getSSN());
Output Preview:
Alice
123-45-6789

5. Abstraction

  • Abstraction hides internal details and exposes only functionality.
  • JS does not have true abstract classes; simulate using base class throwing errors.
Example:

class Vehicle {
  start() { throw new Error("start() must be implemented"); }
}
class Car extends Vehicle {
  start() { console.log("Car started"); }
}
const c = new Car();
c.start();
Output Preview:
Car started

6. Prototype & Prototype Chain

  • Every object has a prototype.
  • Prototype chain allows sharing methods across objects.
Example:

function Animal(name) { this.name = name; }
Animal.prototype.speak = function() { console.log(this.name + " makes a sound"); };
const dog = new Animal("Dog");
dog.speak();
console.log(dog.__proto__ === Animal.prototype);
Output Preview:
Dog makes a sound
true
